# RCC Nexus Protocol - Voynich OS

## Protocol

1. Locate the task.
2. Read the route map.
3. Read the root README and AGENTS.md.
4. Read the target folder README.
5. Preserve claim boundaries.
6. Patch minimally.
7. Run validation.
8. Commit only after validation or explicit limitation disclosure.

## Done criteria

A patch is done only when:

- changed files are known;
- validation is run or explicitly disclosed;
- claim boundaries are preserved;
- folder README and route map remain accurate;
- generated artifacts are audit-visible.

## Non-claim lock

RCC Nexus improves navigation and context integrity. It does not prove code correctness, decipherment, translation, model understanding, patch safety, or production readiness.
