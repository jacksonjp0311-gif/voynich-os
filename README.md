## AI Failure Learning Ledger

| Lesson ID | Failure observed | Root cause | Permanent rule |
|---|---|---|---|
| VOS-L-001 | Initial Git audit showed missing branch/remote information despite a `.git` folder. | Local clone state was incomplete or not fully connected. | Verify remote, branch, latest commit, and status before repo evolution. |
| VOS-L-002 | PowerShell standalone `else` failed when pasted after an `if` block. | Interactive PowerShell cannot accept a separated `else` after a completed `if` block. | Use REPL-safe dual-if patterns or run large patches as `.ps1` files. |
| VOS-L-003 | Large here-string patch was interrupted before writing all files. | Pasting long scripts directly into the REPL is fragile. | For large patches, write a patch script to disk first or keep blocks ASCII-safe and fence-free. |
| VOS-L-004 | JSON tests failed on a valid file with UTF-8 BOM. | Python `utf-8` reader rejected BOM-bearing JSON. | Use `utf-8-sig` for legacy JSON reads and normalize JSON outputs to UTF-8 without BOM. |
| VOS-L-005 | Relative `.NET WriteAllText` path drifted to `C:\Users\jacks`. | `.NET` relative paths can resolve differently than the visible PowerShell prompt. | Use absolute paths rooted at `RepoRoot` for all generated files. |
| VOS-L-006 | README audit reported mojibake after RCC injection. | Prior README contained encoding drift. | Public README must prefer ASCII-safe syntax unless Unicode is intentionally audited. |
| VOS-L-007 | Professional folder renaming was requested before output manifests existed. | Generated artifacts depend on stable paths. | Do not migrate or rename active output folders until manifests and route aliases exist. |
| VOS-L-008 | Public README still opened with overclaiming language after governance sections were appended. | Old README content remained above the professional Nexus mirror. | Replace the public README opening with evidence-bounded framing before publishing. |
